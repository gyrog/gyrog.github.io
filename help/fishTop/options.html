fishTop - Options screen


Version: 3.1.3

"Save file" panel

"Filename" The file used to store the current list. There must always be one of these defined.

"Source folders for collections" panel

"Primary path" The path to the folder that contains the primary collections.
If the slides for hymns are stored in the folder "C:\path\to\Documents\ChurchSlides\Hymns",
and the slides for prayers are stored in the folder "C:\path\to\Documents\ChurchSlides\Prayers",
then "C:\path\to\Documents\ChurchSlides" is the "Primary path", containing 2 collections "Hymns" and "Prayers".
"Enable Secondary" Tick to enable the facility for adding 2 files for each item.
Note: The "Secondary path" must be an existing folder, for this to alter the way that the "Add" button functions.
"Secondary is first" Tick to ensure that the file from the secondary collections, is stored first, and is the one displayed in lists.
If the primary collections are slides, and the secondary collections are music, then this would usually be unchecked.
"Secondary path" The path to the folder that contains the secondary collections.
If the files containing music for hymns are stored in the folder "C:\path\to\Documents\ChurchMusic\Hymns",
and the files containing music for voluntaries are stored in the folder "C:\path\to\Documents\ChurchMusic\Voluntaries",
then "C:\path\to\Documents\ChurchMusic" is the "Secondary path", containing 2 collections "Hymns" and "Voluntaries".
Selecting a file in the music "Hymns" collection could find a corresponding file in the slides "Hymns" collection,
and both get included in the item.
Selecting a file in the music "Voluntaries" collection can never find a corresponding file,
so the item will contain only the music file.
There can only be corresponding files, if there are corresponding collections.
"Mirror Primary and Secondary non-corresponding folders" Click this button to create links between the non-corresponding folders in the Primary and Secondary folders.
This is done so that folders that are unique to Primary or Secondary are always available to the "Add" button.
So, if "Enable Secondary" is ticked, slides in "C:\path\to\Documents\ChurchSlides\Prayers"
will appear in "C:\path\to\Documents\ChurchMusic\Prayers", and can be selected.
Similarly, in normal mode, music in "C:\path\to\Documents\ChurchMusic\Voluntaries"
will appear in "C:\path\to\Documents\ChurchSlides\Voluntaries", and can be selected.
Note: These links are not interpreted as corresponding collections.

"Hint messages" panel

These short help messages, at the top of the screen, provide information on what to do.

"Default" Message to be displayed when there are items available.
"Empty" Message to be displayed when there are no items available.

"Main page appearance" panel

These facilities may be used to improve the appearance of the main page.

"Item font size" Set this to change the size of the text on the buttons. Specifiy the font size in points.
Usually not needed.
"Scroller width" If you get a horizontal scroll bar, set this to add extra pixles to the width.
Usually not needed.

"Executable for Extension" panel

Use these fields to overide the default program used for a file extension.

"Extension" Files with this extension will be processed by the program specified by the "Filename" field.
Examples: "pps" or "ppsx".
"Control dialog" This drop down list allows one of the following options to be chosen:
"None" - The program specified in "Filename" is simply executed.
"WithStopButton" - The specified program is executed, and a dialog is opened containing a "Stop" button.
"PowerPointNavigate" - "Filename" and "Parameter format" are ignored. A dialog is opened to navigate PowerPoint via OLE.
"Filename" This program will be run to process files with the extension specifed in the "Extension" field.
Note: This field must contain the full path to the executable.
"Paramenter format" This contains a formating string for the parameters passed to the program defined in the "Filename" field.
Usually it contains "%s" to represent the data file that is the item.

"App" panel

"Help" Shows this screen.
"Cancel" Closes the screen, ignoring any changes.
"Ok" Closes the screen, saving any changes.
